# Chapter: JavaScript Iteration

## Introduction

Programs become useful when they can process more than one thing. A payroll system must examine every employee record. A browser game must update every active object on the screen. A form validator must inspect each field. A data-cleaning script must transform one value after another. In each of these cases, the program is not solving a single isolated problem. It is performing a repeated action across a sequence, a collection, or a condition. That repeated action is called **iteration**.

In JavaScript, iteration is one of the most fundamental programming ideas. It appears early in the language through classic loop statements such as `for`, `while`, and `do...while`. It also appears in more modern forms, including `for...of`, array methods like `forEach()` and `map()`, and the iterator protocol that underlies many built-in data structures. Beginners often learn iteration first as “doing something over and over,” which is not wrong, but it is incomplete. A stronger understanding requires knowing what is being repeated, when the repetition stops, how variables change during each pass, and what runtime cost the repetition creates.

This chapter develops iteration as both a practical skill and a conceptual model. You will learn the syntax of major JavaScript iteration constructs, the behavior of each one at runtime, the differences between counting loops and collection-based loops, and the tradeoffs among several styles of repeated computation. You will also examine common mistakes, performance implications, and how iteration connects to arrays, strings, objects, and the broader structure of JavaScript programs.

## Foundations

At its core, iteration is the repeated execution of a block of code. Every iteration construct has three essential parts, whether they are visible in the syntax or implied by the language runtime:

1. **A starting state**
   The program begins somewhere: an index at `0`, a counter at `1`, or an iterator positioned before the first element.
2. **A continuation rule**
   The program must know whether another pass should occur. This is usually a Boolean condition, such as `i < 10`.
3. **A state update**
   Something must change so that progress is made. Otherwise, the loop may never terminate.

Here is the general conceptual model:

```javascript
initialize state
while (continuation rule is true) {
    do work
    update state
}
```

**Iteration** is the repeated execution of a block of code, usually controlled by a condition, a counter, or a sequence of values.

### Key Point

A loop is not defined by repetition alone. It is defined by **controlled repetition**.

## Why This Topic Matters

Iteration sits at the center of computer science because real programs rarely operate on just one value. They operate on sets, streams, records, tokens, characters, pixels, nodes, events, and user input histories. If selection lets a program decide, iteration lets a program scale.

In JavaScript specifically, iteration matters because much of the language’s day-to-day work involves arrays, strings, collections, and event-driven processing. Front-end code iterates over DOM elements, form values, and fetched data. Back-end JavaScript iterates through API responses, database results, and object collections. Even when developers are not writing explicit `for` or `while` loops, they are still relying on iteration through methods like `map()` and `filter()`.

A student who learns iteration only as syntax will write code that merely functions. A student who understands iteration as a runtime process will write code that is safer, clearer, and easier to debug.

## Core Concepts

### Repetition with Purpose

Iteration is useful when the same operation must occur multiple times with changing data or changing state. That purpose usually falls into one of several categories:

- counting a fixed number of times
- traversing a sequence
- processing input until a condition changes
- transforming data into a new form
- searching for a value
- accumulating a result

Here is a simple counting loop:

```javascript
for (let i = 1; i <= 5; i++) {
    console.log("Count:", i);
}
```

This loop prints five lines. The variable `i` begins at `1`, continues while `i <= 5`, and increases by one after each iteration.

### Syntax vs. Behavior vs. Meaning

Students often blur three separate ideas:

- **Syntax**: how the code is written
- **Behavior**: what the JavaScript engine actually does
- **Conceptual meaning**: why the code exists and what problem it solves

Consider this loop:

```javascript
for (let i = 0; i < 3; i++) {
    console.log(i);
}
```

**Syntax:** a `for` statement with initialization, condition, and update
**Behavior:** the variable `i` is initialized to `0`, checked against `3`, printed, incremented, checked again, and so on
**Meaning:** the code performs an action once for each integer from `0` through `2`

That separation matters. Many bugs happen because the syntax looks fine, but the behavior does not match the programmer’s intent.

## Terminology

Before going further, you need a precise vocabulary.

A **loop** is a control structure that repeatedly executes a block of code while a condition remains true or while values remain available.

An **iteration variable** is a variable whose value changes from one pass of a loop to the next, such as a counter or index.

**Termination** is the point at which a loop stops executing because its continuation condition becomes false or because control leaves the loop.

An **infinite loop** is a loop that never terminates because its condition never becomes false or its state never updates appropriately.

An **iterable** is an object that can provide values one at a time in sequence. Arrays, strings, maps, and sets are examples of iterable objects in JavaScript.

An **iterator** is an object that produces a sequence of values, typically through repeated calls to its `next()` method.

## JavaScript Loop Structures

## The `while` Loop

A `while` loop repeats as long as its condition is true.

```javascript
let count = 1;

while (count <= 3) {
    console.log("Count is", count);
    count++;
}
```

### Step-by-Step Explanation

1. `count` is initialized to `1`.
2. The condition `count <= 3` is checked.
3. Since it is true, the loop body executes.
4. `"Count is 1"` is printed.
5. `count++` changes `count` to `2`.
6. The condition is checked again.
7. The process repeats until `count` becomes `4`.
8. At that point, the condition is false, so the loop stops.

### Behind-the-Scenes Behavior

A `while` loop checks its condition **before** every iteration. If the condition is false at the start, the body does not execute even once.

### Common Mistake

Forgetting to update the loop variable is the classic way to create an infinite loop.

```javascript
let n = 1;

while (n <= 5) {
    console.log(n);
    // missing n++
}
```

This loop never stops. It is the programming equivalent of stepping onto a treadmill and then bolting your shoes to it.

------

## The `do...while` Loop

A `do...while` loop executes the body first and checks the condition afterward.

```javascript
let number = 1;

do {
    console.log("Number:", number);
    number++;
} while (number <= 3);
```

### Key Point

A `do...while` loop always runs at least once.

This is useful when the program must perform an action before deciding whether to continue, such as reading menu input or prompting a user.

### Comparison Box: `while` vs. `do...while`

| Feature            | `while`                        | `do...while`                           |
| ------------------ | ------------------------------ | -------------------------------------- |
| Condition checked  | Before body                    | After body                             |
| Minimum executions | 0                              | 1                                      |
| Best for           | General conditional repetition | Situations requiring at least one pass |

------

## The Classic `for` Loop

The `for` loop is JavaScript’s most compact counting loop.

```javascript
for (let i = 0; i < 5; i++) {
    console.log("i =", i);
}
```

The general structure is:

```javascript
for (initialization; condition; update) {
    // loop body
}
```

### Syntax / Structure / Model

- **Initialization** runs once before the loop begins.
- **Condition** is checked before each iteration.
- **Update** runs after each iteration.

### Step-by-Step Explanation

For this loop:

```javascript
for (let i = 0; i < 3; i++) {
    console.log(i);
}
```

The engine behaves roughly like this:

```javascript
let i = 0;
while (i < 3) {
    console.log(i);
    i++;
}
```

That is not literally how the source code is rewritten internally, but it is a useful conceptual model.

### Performance Note

For simple counting and indexed array access, the classic `for` loop remains a direct and efficient construct. It also makes control flow very explicit, which helps debugging.

------

## Iterating Through Arrays with Indexes

Arrays are one of the most common reasons to use iteration.

```javascript
const colors = ["red", "green", "blue"];

for (let i = 0; i < colors.length; i++) {
    console.log("Color at index", i, "is", colors[i]);
}
```

### Why This Works

- `colors.length` tells how many elements the array contains
- array indexes start at `0`
- the last valid index is `colors.length - 1`

### Common Mistake: Off-by-One Error

```javascript
const colors = ["red", "green", "blue"];

for (let i = 0; i <= colors.length; i++) {
    console.log(colors[i]);
}
```

This loop uses `<=` instead of `<`. When `i` becomes `3`, `colors[3]` is `undefined` because the valid indexes are `0`, `1`, and `2`.

### Chapter Warning

Most beginner loop errors are not dramatic. They are one character wide.

------

## The `for...of` Loop

JavaScript provides `for...of` for iterating through the values of iterable objects.

```javascript
const colors = ["red", "green", "blue"];

for (const color of colors) {
    console.log(color);
}
```

This loop visits each element in the array directly. No index variable is required unless you specifically need one.

### Why `for...of` Matters

`for...of` is often clearer than an indexed `for` loop when the index itself is not important. It also works with strings, sets, maps, and other iterable objects.

Example with a string:

```javascript
for (const ch of "cat") {
    console.log(ch);
}
```

Output:

```javascript
c
a
t
```

An **iterable object** is any object that can produce values in sequence for constructs like `for...of`.

### Best Practice

Use `for...of` when you want the elements themselves and do not need to manage indexes manually.

------

## The `for...in` Loop

The `for...in` loop is different. It iterates over **property names**.

```javascript
const student = {
    name: "Ava",
    major: "CS",
    level: "Sophomore"
};

for (const key in student) {
    console.log(key, "=>", student[key]);
}
```

Output:

```javascript
name => Ava
major => CS
level => Sophomore
```

### Comparison Box: `for...in` vs. `for...of`

| Loop       | Iterates over           | Common use                  |
| ---------- | ----------------------- | --------------------------- |
| `for...in` | property names (keys)   | objects                     |
| `for...of` | values from an iterable | arrays, strings, sets, maps |

### Common Mistake

Using `for...in` on arrays when you really want the elements.

```javascript
const nums = [10, 20, 30];

for (const x in nums) {
    console.log(x);
}
```

Output:

```javascript
0
1
2
```

That loop prints indexes as strings, not the array values. If you want `10`, `20`, and `30`, use `for...of`.

## Array Iteration Methods

JavaScript arrays provide built-in methods that perform iteration internally. These methods are extremely important in modern JavaScript.

## `forEach()`

`forEach()` executes a function once for each element.

```javascript
const nums = [2, 4, 6];

nums.forEach(function (value, index) {
    console.log("Index:", index, "Value:", value);
});
```

An arrow-function version is common:

```javascript
nums.forEach((value, index) => {
    console.log("Index:", index, "Value:", value);
});
```

### Key Point

`forEach()` is meant for side effects such as printing, updating the DOM, or logging. It does not create a new array.

------

## `map()`

`map()` transforms each element and returns a new array.

```javascript
const nums = [1, 2, 3];
const squares = nums.map(n => n * n);

console.log(squares); // [1, 4, 9]
```

### Conceptual Meaning

“Take each element, apply a transformation, and collect the results.”

### Behind-the-Scenes Behavior

`map()` creates a new array rather than modifying the original one unless your callback explicitly mutates external data.

------

## `filter()`

`filter()` keeps only the elements that pass a test.

```javascript
const nums = [1, 2, 3, 4, 5, 6];
const evens = nums.filter(n => n % 2 === 0);

console.log(evens); // [2, 4, 6]
```

### Conceptual Meaning

“Select the elements that satisfy a condition.”

------

## `reduce()`

`reduce()` combines all elements into one final result.

```javascript
const nums = [1, 2, 3, 4];
const total = nums.reduce((sum, n) => sum + n, 0);

console.log(total); // 10
```

### Step-by-Step Explanation

The callback receives an accumulator and the current value.

- start with accumulator = `0`
- add `1` → accumulator becomes `1`
- add `2` → accumulator becomes `3`
- add `3` → accumulator becomes `6`
- add `4` → accumulator becomes `10`

### Chapter Warning

Students often meet `reduce()` too early and decide JavaScript has chosen violence. The method is powerful, but readability matters. Do not use it just to look clever.

## Demonstration: Three Ways to Sum an Array

Suppose we want to sum the numbers in an array.

```javascript
const values = [5, 10, 15];
```

### Version 1: Classic `for`

```javascript
let total1 = 0;

for (let i = 0; i < values.length; i++) {
    total1 += values[i];
}

console.log(total1);
```

### Version 2: `for...of`

```javascript
let total2 = 0;

for (const value of values) {
    total2 += value;
}

console.log(total2);
```

### Version 3: `reduce()`

```javascript
const total3 = values.reduce((sum, value) => sum + value, 0);

console.log(total3);
```

### Comparison Box

| Style         | Strength                 | Weakness                                |
| ------------- | ------------------------ | --------------------------------------- |
| Classic `for` | explicit control         | more verbose                            |
| `for...of`    | very readable            | no built-in index unless added manually |
| `reduce()`    | concise for accumulation | can be harder to read for beginners     |

### Final Takeaway

These three forms can solve the same problem, but they communicate different things to the reader. Good JavaScript is not just about getting the answer. It is about choosing the form that makes the answer obvious.

## Step-by-Step Example Program

The following example processes student scores.

```javascript
const scores = [82, 91, 76, 88, 95];
let total = 0;

for (const score of scores) {
    total += score;
}

const average = total / scores.length;

console.log("Scores:", scores);
console.log("Total:", total);
console.log("Average:", average);
```

### Step-by-Step Explanation

1. `scores` stores five numeric values.
2. `total` begins at `0`.
3. The `for...of` loop visits each score in turn.
4. Each score is added to `total`.
5. After the loop, `total` equals the sum of all elements.
6. The average is computed by dividing by the number of scores.

### Checkpoint

Why is the division performed **after** the loop rather than during each iteration?

Because the loop’s job is accumulation. The final result is computed once the repeated work is finished.

## Behind-the-Scenes Behavior

To understand iteration well, you need to think like the runtime.

## Loop Conditions Are Re-Evaluated

In `while` and `for` loops, the condition is checked before every iteration. If variables used in the condition change inside the loop, that change affects whether the loop continues.

```javascript
let x = 0;

while (x < 3) {
    console.log(x);
    x++;
}
```

The condition is not checked only once. It is checked every time.

## `for...of` Uses the Iterator Protocol

When JavaScript executes a `for...of` loop, it requests an iterator from the iterable object and repeatedly asks for the next value. Conceptually, the process looks like this:

```javascript
const arr = ["a", "b", "c"];
const iterator = arr[Symbol.iterator]();

console.log(iterator.next()); // { value: "a", done: false }
console.log(iterator.next()); // { value: "b", done: false }
console.log(iterator.next()); // { value: "c", done: false }
console.log(iterator.next()); // { value: undefined, done: true }
```

You do not need to write this manually in an introductory course, but it explains why `for...of` works across many different kinds of data.

### Key Point

Iteration in modern JavaScript is not limited to loop syntax. It is tied to a general protocol for producing values.

## Array Methods Call Functions Repeatedly

Methods like `map()` and `filter()` internally perform iteration. The callback function runs once per element, usually in order from the first index to the last existing element.

This means that when you use `map()`, you are still iterating. You are just delegating the loop mechanics to the method.

## Common Mistakes and Misconceptions

## Mistake 1: Confusing `=` with `===` in Conditions

```javascript
let x = 0;

while (x = 5) {
    console.log(x);
}
```

This is wrong. The expression `x = 5` assigns `5` to `x`, and the value `5` is truthy, so the loop never ends.

Correct version:

```javascript
while (x === 5) {
    console.log(x);
}
```

## Mistake 2: Off-by-One Errors

These happen when a loop runs one time too many or one time too few.

```javascript
for (let i = 1; i < 5; i++) {
    console.log(i);
}
```

This prints `1`, `2`, `3`, `4`. It does **not** print `5`.

## Mistake 3: Using `for...in` for Array Values

Already discussed, but worth repeating because it causes endless confusion.

```javascript
const arr = ["x", "y", "z"];

for (const item in arr) {
    console.log(item); // indexes, not values
}
```

## Mistake 4: Modifying a Collection While Iterating

Consider:

```javascript
const nums = [1, 2, 3, 4];

for (let i = 0; i < nums.length; i++) {
    nums.pop();
}
```

This loop changes the array while also depending on its length. That can produce surprising behavior and skipped elements.

### Chapter Warning

Mutating the same structure you are iterating through is legal in some cases, but it is a fine way to manufacture bugs at industrial scale.

## Mistake 5: Assuming Array Methods Break Like Loops

A `for` loop can use `break` and `continue`.

```javascript
for (const n of [1, 2, 3, 4]) {
    if (n === 3) break;
    console.log(n);
}
```

But `forEach()` does not support `break` in the same way.

```javascript
[1, 2, 3, 4].forEach(n => {
    console.log(n);
    // cannot use break here
});
```

If you need early termination, a regular loop is often better.

## Comparison to Related Concepts

## Iteration vs. Recursion

Both iteration and recursion solve repeated problems.

### Iteration

Repetition is controlled by loop constructs.

```javascript
let total = 0;
for (let i = 1; i <= 4; i++) {
    total += i;
}
```

### Recursion

A function calls itself on a smaller subproblem.

```javascript
function sumTo(n) {
    if (n === 1) return 1;
    return n + sumTo(n - 1);
}
```

### Comparison Box: Iteration vs. Recursion

| Feature          | Iteration                                 | Recursion                           |
| ---------------- | ----------------------------------------- | ----------------------------------- |
| Mechanism        | loops                                     | self-calling functions              |
| State storage    | loop variables                            | call stack                          |
| Common strengths | efficiency, clarity for linear repetition | elegant for recursive structures    |
| Common risk      | infinite loops                            | stack overflow or unclear base case |

For introductory JavaScript, iteration is usually the more direct tool for linear data processing.

## Iteration vs. Selection

Selection chooses among alternatives using `if`, `else`, or `switch`. Iteration repeats work. Many programs combine both.

```javascript
for (const n of [1, 2, 3, 4, 5]) {
    if (n % 2 === 0) {
        console.log(n, "is even");
    }
}
```

The loop provides repetition; the `if` statement provides selection inside each iteration.

## Best Practices

Choose the simplest iteration construct that matches the problem. If you need a counter or index, use a classic `for` loop. If you need the values of an iterable, use `for...of`. If you are transforming arrays into new arrays, prefer `map()`. If you are selecting elements, use `filter()`. If you need side effects such as logging or DOM updates, `forEach()` can be appropriate. If you need early exit, a plain loop is often clearer than forcing the problem into an array method that was not designed for it.

Keep loop conditions easy to read. A correct loop with a murky condition is still bad code. Use meaningful variable names. `i` and `j` are acceptable for short index-based loops, but names like `score`, `student`, or `item` are better when they communicate the data being processed. Avoid mutating collections during traversal unless you have a very good reason and understand the consequences. Prefer clarity over cleverness. A one-line `reduce()` masterpiece that nobody can read is not impressive. It is just a future bug with good posture.

### Debugging Note

When a loop misbehaves, inspect three things first: the starting value, the stopping condition, and the update step. Most loop bugs live in one of those three places.

### Performance Note

Big-O analysis matters more than micro-obsessing over which loop keyword is “fastest.” A linear pass through an array is usually `O(n)`. Two nested passes can become `O(n^2)`. That matters far more than shaving off a tiny constant factor by arguing about loop style on the internet.

## Chapter Summary

Iteration is the mechanism by which JavaScript performs repeated work. It appears in traditional loops such as `while`, `do...while`, and `for`, as well as in collection-oriented forms such as `for...of` and array methods like `forEach()`, `map()`, `filter()`, and `reduce()`. Although these constructs differ in syntax, they all rely on the same essential ideas: a starting state, a rule for continuing, and a change in state that moves the computation forward.

A strong understanding of iteration requires more than memorizing syntax. You must be able to predict runtime behavior, including when conditions are checked, how variables change, how data is traversed, and when loops terminate. You must also distinguish between different iteration tools. `for...of` is for iterable values; `for...in` is for property names. `map()` transforms data; `filter()` selects data; `reduce()` combines data. The best choice depends on the problem being solved.

Finally, good iteration is disciplined. Most loop bugs come from poor control over initialization, conditions, and updates. Clear loop structure, careful choice of construct, and respect for readability will produce code that is not only correct, but maintainable. That is the real goal. Computers will run ugly loops. Humans have to live with them.

## End-of-Chapter Glossary

**accumulation**
The process of combining multiple values into one result, such as summing numbers in an array.

**callback**
A function passed as an argument to another function, often used in array iteration methods like `map()` or `forEach()`.

**counter**
A variable used to track the number of times a loop has executed.

**filtering**
The process of selecting only the elements that satisfy a specified condition.

**infinite loop**
A loop that never stops because its termination condition is never reached.

**index**
The numeric position of an element in an array. JavaScript array indexing starts at `0`.

**iterable**
An object capable of producing its values one at a time in sequence, such as an array or string.

**iteration**
The repeated execution of a block of code.

**iteration variable**
A variable whose value changes during repetition, such as an index or loop counter.

**iterator**
An object that provides values sequentially, usually through a `next()` method.

**loop**
A control structure that repeats code based on a condition or sequence.

**off-by-one error**
A logic error in which a loop executes one time too many or one time too few.

**side effect**
A change caused by code beyond simply returning a value, such as printing output, modifying the DOM, or changing external data.

**termination**
The stopping point of a loop when its continuation rule becomes false or execution leaves the loop.

**transformation**
The process of converting each element of a collection into a corresponding new value.

**traversal**
The act of visiting elements in a collection one by one.
