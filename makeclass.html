Create A Trigger Box with NiagraSystem and a universal counter

Example Overview

1. Trigger Event: K2Node_Event

  • Purpose: The graph starts with a ReceiveActorBeginOverlap event, triggered when an actor overlaps with another.

  • Nodes:

    • The K2Node_Event_1 handles the overlap event.

    • Outputs from this node include the overlapping actor (OtherActor) and execution (then) pins for chaining further logic.

2. Spawning a Niagara Particle System

  • Node: K2Node_CallFunction_1

  • Function: Calls NiagaraFunctionLibrary::SpawnSystemAtLocation, which spawns a particle effect.

  • Details:

    • SystemTemplate: Specifies the Niagara system (e.g., explosion effect).

    • Location: Retrieves the location of the box component in the scene.

    • bAutoDestroy: Automatically destroys the particle effect once complete.

  • This node is connected to the then pin of the event node.

3. Playing a Sound

  • Node: K2Node_CallFunction_3

  • Function: Plays a 2D sound using GameplayStatics::PlaySound2D.

  • Details:

    • The sound asset is specified (e.g., an explosion sound).

    • Configures volume and pitch multipliers.

4. Destroying the Actor

  • Node: K2Node_CallFunction_6

  • Function: Calls K2_DestroyActor, destroying the actor after the sound and particle effect.

5. Delay Before Spawning New Actor

  • Node: K2Node_CallFunction_15

  • Function: Adds a delay (e.g., 1 second) using KismetSystemLibrary::Delay.

6. Spawning a New Actor

  • Node: K2Node_SpawnActorFromClass

  • Function: Spawns a new actor using the specified class and transform.

  • Details:

    • Class: Specifies the class of the actor to spawn (e.g., BP_ThirdPersonCharacter).

    • SpawnTransform: Specifies the spawn location, rotation, and scale.

7. Possessing the Spawned Actor

  • Node: K2Node_CallFunction_14

  • Function: A controller takes control of the spawned actor using Controller::Possess.

8. Iterating Through Actors

  • Node: K2Node_CallFunction_29

  • Function: Gets all actors of a specified class using GameplayStatics::GetAllActorsOfClass.

  • Details:

    • The retrieved actors are processed using a ForEachLoop.

9. Text Render Component Updates

  • Node: K2Node_CallFunction_36

  • Function: Updates the text of a TextRenderComponent to display a new value.

  • Details:

    • Converts an integer to text using KismetTextLibrary::Conv_IntToText.

    • The text is set using TextRenderComponent::K2_SetText.

10. Game Instance Interaction

  • Node: K2Node_CallFunction_44 and K2Node_DynamicCast_10

  • Function: Accesses a custom Game Instance to retrieve or modify variables like xxTotalLosses.

Summary of Workflow

  1. Trigger: An actor begins overlap, triggering effects.

  2. Visuals: A Niagara system is spawned at the location.

  3. Sound: A sound plays concurrently with the particle effect.

  4. Cleanup: The overlapping actor is destroyed.

  5. Delay and Respawn: A new actor is spawned and controlled after a delay.

  6. Actor Iteration: All actors of a specific class are processed for updates.

  7. UI Updates: TextRender components are dynamically updated.

This represents a procedural event-driven system often used in Unreal Engine games to handle interactions, effects, and gameplay updates.

Create a game instance class. Here you can create global variables that all the levels can access.

  • In the browser contents, right click choose new blueprint class.

  • In the all classes options, choose new game instance and call it myGameInstance.

  • Open the project settings and replace the default game instance with the one created above - myGameInstance

 

FYI

Unreal Engine sharing blueprints platforms:

  • Klee

    A web-based tool that lets you copy and paste blueprints from Unreal Engine into a browser.

    You can then share the blueprints on your website.

     

  • Blueprint Viewer

    A Chrome extension that allows you to read .bp files in a GIT repository, including Unreal Engine blueprints.

     

  • BlueprintUE

    An unofficial tool that allows developers to share blueprints by copying them, getting a shareable link, and studying individual elements.

     

  • Visual Studio

    Allows you to view Unreal Engine blueprints in Visual Studio.

    You can double-click a Blueprint to open the Blueprint asset viewer and view its properties.

     

You can also use Blueprint Interfaces to share data between different Blueprints. A Blueprint Interface is a collection of functions that can be added to other Blueprints.
